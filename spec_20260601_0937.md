# 스펙 — prism-fs (EARS 원칙)

> 작성: 2026-06-01 09:37 · 버전 0.1.0 · 소유자: 신한금융지주 회계팀
> 표기: SHALL=필수. 각 요구사항은 검증 가능(testable)하게 기술.

---

## Ubiquitous (상시)

- **U1** The system SHALL 4대 금융지주(신한·KB·하나·우리)의 연결(CFS)·별도(OFS) 재무제표와 주석을 `회사 × 기간 × {연결,별도} × {재무제표,주석}` 차원으로 보관·조회한다.
- **U2** The system SHALL DART 원문 금액(`thstrm_amount`·`frmtrm_amount`·`frmtrm_q_amount`·`bfefrmtrm_amount`)을 문자열 그대로 보존하며 환산·재구성하지 않는다.
- **U3** The system SHALL 모든 파생값(증감·차이·비율)에 계산식·입력 account_id·원문값·`engine:"deterministic"`을 provenance로 첨부한다.
- **U4** The system SHALL 모든 외부 자산(CSS·pdf.js·임베딩 모델)을 로컬에 동봉하여 외부 CDN 호출 없이 동작한다.
- **U5** The system SHALL 로그·응답·예외 메시지에서 `DART_API_KEY`·`OPENAI_API_KEY`·`sk-*`·`Bearer` 토큰을 마스킹한다.
- **U6** The system SHALL 재무제표 비교 시 계정을 IFRS `account_id` 기준으로 매칭한다(표기명 불일치 흡수).
- **U7** The system SHALL 원문 수치 라인과 파생(보조) 라인을 시각적으로 구분하여 표시한다.

## Event-driven (WHEN)

- **E1** WHEN 사용자가 두 기간 이상 셀을 선택하면 THE system SHALL 동일 account_id의 증감액·증감률을 보조 라인으로 표시한다.
- **E2** WHEN 손익(CIS) 계정의 증감을 계산하면 THE system SHALL 비교 컬럼으로 `frmtrm_q_amount`(전기 동기)를, 재무상태표(BS) 계정은 `frmtrm_amount`(전기말)를 자동 선택한다.
- **E3** WHEN 사용자가 연결·별도 비교를 요청하면 THE system SHALL `CFS − OFS` 차이를 "자회사효과(추정)" 라벨과 함께 보조 라인으로 표시한다.
- **E4** WHEN 주석 비교 대상이 선택되면 THE system SHALL 동일 fs_div(연결↔연결, 별도↔별도)끼리만 매칭한다.
- **E5** WHEN 사용자가 주제(자연어/키워드)를 입력하면 THE system SHALL 임베딩+BM25 하이브리드로 매칭하고 점수·lexical_hit을 함께 표시한다.
- **E6** WHEN 4사 벤치마킹을 요청하면 THE system SHALL 동일 기간·fs_div의 동일 account_id를 4사 나란히 표시하고 순위를 매긴다.
- **E7** WHEN 구조비율을 요청하면 THE system SHALL 분자·분모가 모두 원문에 존재할 때만 계산하고 계산식을 동봉한다.

## Unwanted (IF-THEN)

- **X1** IF 비교 셀들의 표기 단위(detected_unit)가 다르면 THEN THE system SHALL 단위 불일치 경고를 표시하고 자동 환산하지 않는다.
- **X2** IF 증감률 또는 비율의 분모가 0이면 THEN THE system SHALL "N/A"를 표시하고 계산하지 않는다.
- **X3** IF 연결·별도 비교에서 account_id가 한쪽에만 존재하면 THEN THE system SHALL "대응 없음"을 플래그하고 차감하지 않는다.
- **X4** IF 외부 LLM 키(`OPENAI_API_KEY`)가 없으면 THEN THE system SHALL LLM 기능을 비활성하고 결정론 기능만 제공한다.
- **X5** IF `bfefrmtrm_amount`가 비어 있으면 THEN THE system SHALL 단일 셀로 전전기 비교를 추론하지 않고 여러 기간 셀 교차 로드를 요구한다.

## State-driven (WHILE)

- **S1** WHILE 임베딩 모델이 로컬에 부재하면 THE system SHALL bigram 폴백으로 동작하고 정확도 저하를 배너로 고지한다.
- **S2** WHILE 검토보고서(review, slim)만 적재된 셀이면 THE system SHALL 별도 주석 요청 시 미지원 사유를 표시한다.

## Optional (WHERE)

- **O1** WHERE 사용자가 회계정책(주기) 필터를 켜면 THE system SHALL 회계정책 키워드 룰셋으로 태깅된 주석만 노출한다.
- **O2** WHERE `OPENAI_API_KEY` 또는 로컬 Ollama가 활성이면 THE system SHALL 주석 요약을 제공하되 근거 페이지를 함께 표시한다.
