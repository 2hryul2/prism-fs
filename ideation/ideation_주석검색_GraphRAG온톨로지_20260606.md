## 1. 배경·목표

prism-fs의 본래 목적은 **4대 금융지주 주석 검색·비교**다. 검색 파이프라인(임베딩+BM25 하이브리드·
note_kind 가중·동의어·IDF 제목보너스)은 이미 성숙해 골든셋에서 `hit@1=1.0`을 찍지만, 두 가지가 비어 있다:
**(1) 측정 신뢰성**과 **(2) 검색으로 얻은 지식의 축적·되먹임**.

이 문서는 Obsidian/Karpathy의 *interlinked-notes*(백링크로 지식을 연결·시각화) 패턴을 회계 도메인에
적용해, **검색으로 발견한 개념 관계를 `[[ ]]`형 개념 그래프로 축적(지식화)하고, 그 그래프가 검색을
개선하는 선순환**을 설계한다. 두 목표는 사실 하나로 묶인다.

### 확정 요구 2가지
1. **주석 검색 정확도 향상** — 회계 도메인 개념·동의어·약어를 반영해 재현율·정밀도를 함께 끌어올림
2. **검색한 내용의 지식화** — 검색·검토 과정에서 확인된 개념 관계를 그래프로 축적 → 검색에 되먹임

### 통합 그림 (선순환)
```
검색 → [검토시점 사용자 의견 캡처] → 검수 게이트(pending→사람확정→active)
        → ┬ 골든셋(측정 신뢰 확보) ┬→ 검색 개선
          └ 개념 그래프(지식화)    ┘
        ↑──────────── eval --probe 회귀 가드 ────────────────┘
```

## 3. 설계 — 회계개념 지식 그래프 + HITL 선순환

### 3.1 확정 결정
| 항목 | 결정 | 근거 |
|---|---|---|
| 피드백 신호 | **explicit only**(👍/👎·정답지정·관계제안) | 고신뢰·저잡음, 안전경계(추론 없음) |
| 그래프 저장 | **신규 `concept_graph.json`** (동의어층 `synonyms_data.json` 분리·하위호환) | 관심사 분리, 기존 `_load_groups`/`expand_query` 무수정 |
| 관계 어휘 | **풍부**: 동의어·관련·상위·하위·측정대상·공시영역 | 회계팀=도메인 전문가, 장기 표현력 |
| 쿼리 확장 | **동의어만 BM25 자동병합**, 관련/상위/하위/도메인=추천·내비게이션만 | 오확장 차단(검수표 C "병합금지" 정합), 정밀도 우선 |
| 하위(narrower) | **추천만**(자동 재현율 확장 안 함) | prism-fs 정밀도 우선 철학 |
| 축적 경로 | 검색 검토시점 explicit 캡처 → **검수 게이트(pending→사람확정→active)** | 자동반영 금지(안전경계 + Karpathy 능동 큐레이션) |

### 3.2 concept_graph.json 스키마
```json
{ "schema": 1, "nodes": [{
  "id": "ecl",
  "canonical": "대손충당금",
  "aliases": ["신용손실충당금","기대신용손실","ECL","대손준비금"],
  "links": [
    {"target": "신용위험",  "relation": "관련",     "confidence": "verified"},
    {"target": "FVOCI",     "relation": "측정대상", "confidence": "verified"},
    {"target": "회계정책",  "relation": "공시영역"}
  ],
  "status": "active",                 // active | pending (검수 게이트)
  "source": "K-IFRS 9", "verified_by": "신한 회계팀", "verified_at": "2026-..."
}]}
```
- **동의어**는 노드 내부 `aliases`(치환 가능 → 쿼리 자동병합). **개념 간 엣지**는 `links`(관련/상위/측정대상/공시영역).
- 동의어층 `synonyms_data.json`은 그대로 유지. `expand_query` = 기존 동의어 그룹 ∪ 그래프 active aliases.

### 3.3 관계 종류별 검색 동작 (정밀도 우선)
| relation | 의미 | 검색 동작 |
|---|---|---|
| 동의어 synonym | 치환 가능 동일 개념 | **BM25 쿼리 자동병합** (유일) |
| 관련 related | 인접·연관 | `suggest_terms` 칩 (추천만) |
| 상위 broader | 포함하는 상위 개념 | 내비게이션 + 칩 |
| 하위 narrower | 포함되는 하위 개념 | 추천만 (자동확장 X) |
| 측정대상 measures | A가 B로 측정/평가 | 칩 |
| 공시영역 disclosed-in | A가 B 영역에 공시 | 칩 |

## 4. 안전경계 정합 (불변 원칙 유지)
| 안전경계 | 본 설계에서의 구현 |
|---|---|
| AI 무경유·결정론 | 그래프 traversal·동의어 병합 전부 결정론. LLM 미사용 |
| 오확장 차단 | **동의어만 쿼리 병합**, 관련/상위/하위/도메인은 추천만 |
| 사람 검증(HITL) | 피드백 → pending → **검수 게이트** 통과해야 active (자동반영 금지) |
| provenance | 검색 결과에 매칭 경로·관계 타입·신뢰도·`engine:deterministic` 동봉 |
| 회귀 방지 | 각 단계 반영 전후 `eval_search --probe`로 hit@k·MRR·교차변형 비교(A-2/B-3 막은 루프 재사용) |
| 폐쇄망 | 외부 CDN/네트워크 0, 데이터 임베드·vendored |

## 6. 구현 검토 및 계획 (phasing — 미착수)

> 원칙: 각 Phase는 `eval_search --probe` 회귀 게이트 통과가 합격선. 데이터(graph/feedback JSON)는
> storage/산출물 — 코드만 커밋(기존 관례). 검수 도구는 `scripts/curate_index_claude.py` 패턴 미러.

| Phase | 대상·변경 | 의존 | 위험 | 검증 | 규모 |
|---|---|---|---|---|---|
| **P1 토대** | 신규 `src/concept_graph.json` + 로더 `src/concept_graph.py`. 기존 `synonyms_data.json` 29그룹을 그래프 노드로 **무손실 마이그레이션**(canonical+aliases·status=active·source=검수표). 검색 동작 변화 0 | 없음 | 낮음 | 마이그레이션 무손실·로더 폴백 단위테스트, **pytest 63 유지** | S |
| **P2 되먹임 읽기경로** | `expand_query`에 그래프 active aliases 병합(동의어만)·`suggest_terms`에 관계 칩(관련/상위/하위/도메인)·**coverage·topic-map에 동의어 확장(누락 보완)** | P1 | 중 | `eval --probe` 무회귀 + coverage 경로 이득 측정 | M |
| **P3 피드백 캡처** | compare top-5·RAG top-k UI에 👍/👎·정답지정·관계제안 → 신규 `POST /api/feedback` → `storage/search_feedback.json`(pending). **캡처만, 검색 동작 불변** | P1 | 낮음 | 피드백 영속·스키마 검증 | M |
| **P4 검수 게이트+승격** | 큐레이션 도구(`scripts/curate_feedback.py`, validate/diff/apply+.bak): pending 검토 → `concept_graph` active edge / `golden_search.json` 항목 **승격(사람 확정)** | P1·P3 | 중 | 승격 반영·골든셋 성장·**확대셋 재측정** | M |
| **P5 운영** | pending 큐 주기 리뷰·고아 개념 정리·골든셋=회계팀 확정셋. 확대셋 기준 정확도 측정 | P4 | 낮음 | 운영 지표·정기 회귀 | S |

### 6.1 시퀀싱 옵션 (P1 이후)
- **가치 우선**: P1 → **P2**(coverage 동의어 보완 = 즉시 측정가능 이득) → P3 → P4
- **측정 우선**: P1 → **P3·P4**(피드백+검수+골든셋 성장으로 측정 신뢰 먼저) → P2
- 데모를 P1 산출 `concept_graph.json`에 연결하면 시연 신뢰도도 동반 상승(손제작 → 실데이터).

## 7. 미결정·리스크·다음 결정점
- **시퀀싱**: P1 후 가치 우선(P2) vs 측정 우선(P3·P4) — 미결정.
- **관계 어휘 운영 부담**: 풍부 관계어휘 채택 → 회계팀 피드백 입력 부담. 초기엔 동의어·관련 위주로
  시작하고 도메인 관계는 점증 검수 권장.
- **골든셋 확정**: placeholder → 회계팀 확정셋 확대가 측정 신뢰의 전제(P3·P4 산물).
- **명칭 일관성**: 동의어 변형으로 인한 노드 분열 방지 — canonical+aliases 규율, 생성 전 기존 노드 검색.
- **PoC ROI 연결**: 설명가능성·provenance = *리스크 감소*, 검색 정확도·관련어 추천 = *효율성* 카테고리 기여.

## 8. 요약
검색 정확도와 지식화는 **하나의 선순환**으로 묶인다: 검색 검토시점의 explicit 피드백 → 검수 게이트 →
개념 그래프(지식화) + 골든셋(측정) → 검색 개선. 안전경계(동의어만 병합·HITL 검수·provenance·회귀가드)는
전부 유지된다. 프로토타입(`concept_graph_demo.html`)으로 "설명 가능한 graph-RAG"를 시연 가능하며,
실구현은 P1(무위험 토대)부터 단계적으로, 각 단계 `eval --probe` 게이트로 회귀를 차단하며 진행한다.
