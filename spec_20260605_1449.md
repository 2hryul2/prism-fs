# 스펙 — 검토보고서(review) 라이브러리 1급 노출 + 비교·질의 (EARS 원칙)

> 작성: 2026-06-05 14:49 · 대상 버전 0.2.0 · 소유자: 신한금융지주 회계팀
> 연계: ideation_검토보고서라이브러리추가_20260605_1446.md
> 표기: SHALL=필수. 각 요구사항은 검증 가능(testable)하게 기술.
> 범위: 문서축(doc_type) = {report=분기보고서, review=분기검토보고서}를 1급 선택 차원으로 추가.

---

## Ubiquitous (상시)

- **RU1** The system SHALL 라이브러리·비교·질의의 선택 단위를 `회사 × 기간 × 문서(report|review) × fs_div(연결|별도)` 4차원으로 식별한다.
- **RU2** The system SHALL 분기보고서(report)와 검토보고서(review)를 라이브러리 셀에서 **둘 다** 선택 가능한 항목으로 노출한다.
- **RU3** The system SHALL 검토보고서(review, slim)를 연결 단일 축으로 취급한다(별도 미제공).
- **RU4** The system SHALL 비교·질의 결과·PDF 패널에 문서 출처(📄 분기보고서 / 📋 검토보고서)와 fs_div를 함께 라벨로 표기한다.
- **RU5** The system SHALL 동일 문서축·동일 fs_div끼리만 비교 매칭한다(report연결↔report연결, review연결↔review연결).

## Event-driven (WHEN)

- **RE1** WHEN 사용자가 라이브러리 셀에서 검토보고서(연결) pill을 선택하면 THE system SHALL 해당 항목을 `회사/기간#review#연결` 키로 선택 목록에 추가한다.
- **RE2** WHEN 검토보고서 대상으로 비교조회를 요청하면 THE system SHALL `doc_type=review` 인덱스(index_review.json)에서 주석을 매칭하고 review.pdf의 해당 페이지를 표시한다.
- **RE3** WHEN 검토보고서 대상으로 자연어질의(RAG)를 요청하면 THE system SHALL `doc_type=review` 주석만 근거로 검색·인용한다.
- **RE4** WHEN 사용자가 RAG 탭에서 문서 소스 토글을 전환하면 THE system SHALL 선택된 doc_type을 질의 파라미터로 전달한다.
- **RE5** WHEN 커버리지·주제맵·XBRL 탭에서 문서축을 전환하면 THE system SHALL 해당 doc_type 인덱스로 집계를 재계산한다.
- **RE6** WHEN PDF 패널을 렌더하면 THE system SHALL 선택 항목의 doc_type에 맞는 PDF(report.pdf 또는 review.pdf)를 로드한다.

## Unwanted (IF-THEN)

- **RX1** IF 검토보고서 대상에 별도(fs_div=별도) 주석을 요청하면 THEN THE system SHALL "검토보고서는 연결 주석만 제공합니다"를 표시하고 빈 결과를 정상 처리한다(오류 아님).
- **RX2** IF 셀에 검토보고서가 미수집(`review_indexed=false`)이면 THEN THE system SHALL 검토보고서 pill을 비활성(–) 처리한다.
- **RX3** IF API 호출에 doc_type이 생략되면 THEN THE system SHALL 기존과 동일하게 `report`로 처리한다(하위호환).
- **RX4** IF report연결과 review연결을 동시에 비교 대상으로 선택하면 THEN THE system SHALL 두 문서를 각각 별개 패널로 표시하고 혼합 매칭하지 않는다.

## State-driven (WHILE)

- **RS1** WHILE 검토보고서(slim)만 적재된 셀이면 THE system SHALL 별도 주석 요청 시 미지원 사유를 표시한다. (기존 S2 유지)
- **RS2** WHILE 분기보고서에 별도 주석이 추출되어 있으면 THE system SHALL "별도 주석 미추출" 안내문을 표시하지 않는다(stale 문구 제거).

## Optional (WHERE)

- **RO1** WHERE 사용자가 분기보고서와 검토보고서를 같은 기간에 동시 선택하면 THE system SHALL 두 문서의 동일 주제 주석을 나란히 대조할 수 있게 한다(연결 동일 축).
