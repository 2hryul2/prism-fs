# 코드 구조 분석 — prism-fs · 2026-06-06

> 4대 금융지주 주석·재무제표 비교 대시보드의 백엔드(FastAPI) 코드 구조 정리.
> 코드 직접 분석 기반. 안전경계(숫자 AI 무경유·결정론·인용강제·외부CDN 0·키마스킹) 전반에 관통.

## 1. 한눈에 보기

```
run_server.py (진입점)
      │  uvicorn :8021 + 브라우저 자동오픈 + 오프라인 강제(HF_HUB_OFFLINE)
      ▼
   app.py  ─── FastAPI 메인 허브 (2046줄, 32개 엔드포인트)
      │
      ├─ paths.py          경로 중앙화 (dev / PyInstaller frozen 일관)
      ├─ collect_dart.py   DART OpenAPI 수집 (1335줄)
      ├─ fs_compare.py     재무제표 결정론 비교 + provenance
      ├─ notes_rag.py      주석 RAG (정성 텍스트, 숫자 무경유)
      ├─ note_filters.py   주석 종류 분류 (주기/서술형)
      ├─ note_topics.py    표준 주제 매핑
      ├─ safety.py         시크릿 마스킹 + provenance 표준형
      ├─ synonyms.py       동의어 쿼리 확장 (BM25/lexical)
      └─ xbrl_tagging.py   XBRL 상세태깅 진단 (L1/L2/L3, 709줄)
```

**설계 원칙**
- **app.py 중심 허브**: 모든 라우트가 app.py에 모이고, 도메인 로직은 모듈로 분리.
- **결정론 모듈 격리**: `fs_compare`·`note_filters`·`note_topics`·`safety`·`xbrl_tagging`은 AI 무경유 순수 규칙/집계.
- **경로 일관성**: `paths.py`가 dev/frozen 차이를 흡수 → 모든 storage 경로가 단일 소스.
- **선택적 의존성**: `fitz`·`httpx`·`OpenDartReader`·`kiwipiepy`·`sentence-transformers` 미설치 시 해당 기능만 스킵, 메인 흐름 무회귀.

## 3. FastAPI 엔드포인트 (32개)

**라이브러리 관리** — `POST /api/library/upload` · `GET /api/library` · `GET·DELETE /api/library/{company}/{period}`
**인덱싱** — `POST /api/library/index/{company}/{period}` · `GET /api/library/index/status`
**DART 수집** — `POST /api/library/collect` · `GET /api/library/collect/status`
**비교조회** — `POST /api/compare` (topic/number 모드, top-5 후보·동의어 제안)
**재무제표(fs_compare 래핑)** — `GET /api/fs/`: `accounts` · `delta` · `consolidated-vs-separate` · `benchmark` · `ratio` · `timeseries` · `timeseries-accounts` · `flags` · `consolidated-subtotals`
**신한 인사이트** — `POST /api/coverage`(주제×회사 커버리지) · `POST /api/structure-diff`(신한 vs 동종 차집합)
**주석 정성분석** — `GET /api/notes/`: `account-refs` · `topic-map` · `compare-memo`(LLM 옵트인) · `rag`(인용강제)
**XBRL 진단** — `GET /api/xbrl-tagging` · `GET /api/xbrl-tagging/matrix`
**유틸** — `GET /api/terms/suggest` · `GET /api/pdf`(스트리밍) · `GET /api/health`

## 6. 검색 파이프라인 (요약)

1. **인덱싱**: 주석을 page-정확 청크(≈450자·overlap80·페이지당≤3·노트당≤40)로 분할 → 청크별 임베딩 + kiwi 형태소 BM25 토큰. `index.json schema=2`.
2. **질의**: 임베딩(원질의) + BM25(동의어 확장) 하이브리드. note_kind별 가중(주기 0.85cos/0.15bm·완화 임계 0.40 / 서술형 0.7/0.3·0.45). IDF 가중 제목 토큰 보너스(유사 정책주석 변별).
3. **채택**: 점수 플로어 + lexical_hit 두 신호. `match_page`=최고 유닛 실제 페이지 → RAG/메모 인용 정밀화.
4. **평가**: `scripts/eval_search.py` 골든셋 25문항 → hit@1=1.0·MRR=1.0 (2026-06-04 기준). C-1 평가 루프가 회귀 사전 차단(A-2/B-3은 측정 후 롤백).
