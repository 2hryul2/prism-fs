## 2026-06-08 10:10 작업 내역 (버그수정: DART 수집 라이브러리 미등록)

- **증상**: DART 수집(예: 신한 2025FY 사업보고서) 후 라이브러리 매트릭스에 등록되지 않음(빈 "+업로드").
- **원인 (재현·확인)**:
  1. 카탈로그 등록이 **인덱싱 성공에 종속** — `_collect_and_index` 는 report/review 인덱싱이 돼야만 `embed_and_write_index`→`upsert_catalog_entry` 로 등록. 본문 PDF 미확보(annual report 표시 PDF 다운로드 실패 → `display_pdf=manual_upload_required`)면 인덱싱 0건 → **미등록**. 단, 재무데이터(XBRL 7개·`fs_structured.json` 385KB)는 정상 수집됨.
  2. 인덱싱 가드가 불안정한 `report_collected` 플래그에 의존 — report.pdf 가 **디스크에 있어도** 플래그 False 면 건너뜀(실측: 신한 2025Q2/2026Q1 등). **review_sep 인덱싱은 아예 누락**.
- **수정**:
  - `_collect_and_index`: ① 수집 직후 `upsert_catalog_entry` 로 **무조건 등록**(인덱싱과 분리, `fs_collected`/`*_collected`/`rcept_no`/`report_nm` 기록, 기존 행 머지 보존). ② 인덱싱을 **디스크 실제 PDF 기준** 루프로 변경(`report/review/review_sep` 존재 시 인덱싱) → review_sep 누락·플래그 불일치 해소. ③ done/indexing 상태에 `review_sep_collected`/`fs_collected` 포함.
  - `index.html renderMatrix`: 셀 채움 기준을 3문서 인덱싱·진행·**수집됨(collectedOnly)** 으로 확장. 본문 PDF 없이 재무데이터만 수집된 셀은 "📊 재무데이터 수집됨 · 본문 PDF 필요" 캡션으로 등록 표시(빈 셀과 구분). report 진행률 pill 은 `st0` undefined 가드 추가.
