## 0. 한눈에 — 패턴 지도 & 조합 레시피

| ID | 패턴 | 한 줄 |
|---|---|---|
| **P1** | 개념 온톨로지 레이어링 | 평면 동의어 위에 그래프(노드+타입드 엣지+메타)를 하위호환 레이어로 얹기 |
| **P2** | 타입드 관계 + 차등 검색 동작 | 치환 가능한 것(동의어)만 쿼리 병합, 인접한 것은 추천만 → 오확장 차단 |
| **P3** | 설명 가능한 Graph-RAG | 질의→개념→타입드 엣지→문서 경로 + provenance로 "왜 이 결과인가" 설명 |
| **P4** | HITL 피드백 선순환 + 검수 게이트 | explicit 피드백→pending→사람 확정→active. 지식·평가셋 동시 성장 |
| **P5** | 측정 우선 + 회귀 게이트 | 골든셋·eval 프로브를 변경 전후 비교, 회귀는 거부 |
| **P6** | 정밀도-보존 쿼리 확장 | lexical/BM25만 동의어 확장, 임베딩은 원질의 유지 |
| **P7** | 핫패스 분리 (사전계산→JSON) | 무거운 파싱·임베딩은 오프라인 선계산, 런타임은 JSON만 읽음 |
| **P8** | 폐쇄망 친화 그래프 시각화 | 무의존 force 레이아웃(vanilla JS/SVG)·데이터 임베드·외부 CDN 0 |

**조합 레시피 (선순환)**: `P3 검색(설명가능) → P4 검토시점 피드백 캡처 → 검수 게이트 →
P1 그래프(지식화) + 골든셋 → P2/P6 되먹임으로 검색 개선 → P5 회귀 게이트로 안전 확정`.

## P1 — 개념 온톨로지 레이어링

**문제**: 평면 동의어 리스트(`["A","B",...]`)는 개념 간 관계·근거·출처를 표현 못 하고, 검색 로직을
새로 쓰면 기존 동작이 깨질 위험.

**기법**: 평면 동의어층을 **유지**한 채, 그 위에 그래프 레이어를 얹는다.
- 노드 = `{ canonical, aliases[], links[], status, source, verified_by }`
- 엣지(`links`) = 개념 간 **타입드 관계**. 동의어는 노드 내부 `aliases`(치환 가능).
- 로더는 그래프가 있으면 사용, 없으면 평면 동의어로 **폴백**(하위호환).

```json
{ "schema": 1, "nodes": [{
  "id": "ecl", "canonical": "대손충당금",
  "aliases": ["신용손실충당금","ECL"],
  "links": [{"target":"신용위험","relation":"관련","confidence":"verified"}],
  "status": "active", "source": "K-IFRS 9", "verified_by": "도메인팀"
}]}
```

**함정**: 명칭 표류(동일 개념을 다르게 표기) → 노드 분열. → `canonical + aliases` 규율 + **생성 전
기존 노드 검색**. status(active/pending)로 미검증 노드 격리.

**재사용 조건**: 동의어/용어집이 이미 있고, 관계·출처를 점진적으로 얹고 싶을 때. 기존 검색을
무수정 유지해야 하는 운영 시스템에 특히 적합.

**prism-fs 적용**: `synonyms_data.json`(평면 29그룹) 유지 + 신규 `concept_graph.json`. `_load_groups`
무수정.

## P2 — 타입드 관계 + 차등 검색 동작 (핵심)

**문제**: 관계 어휘를 풍부하게 두면(상위/하위/관련/도메인) 표현력은 오르지만, **모든 관계로 쿼리를
확장하면 재현율 노이즈(오확장)** 가 폭증.

**기법**: 관계를 분류하고 **관계 타입별로 검색 동작을 다르게** 둔다. 핵심 원칙 = *치환 가능한 것만
쿼리에 병합, 인접한 것은 추천만*.

| 관계 | 의미 | 검색 동작 |
|---|---|---|
| 동의어 | 치환 가능 동일 개념 | **쿼리 자동 병합** (유일) |
| 관련 | 인접·연관 | 추천 칩만 |
| 상위/하위 | 개념 계층 | 내비게이션 + 추천 |
| 도메인 관계(측정대상·공시영역 등) | 의미적 연결 | 추천 칩만 |

**함정**: "풍부한 그래프 = 풍부한 자동확장" 유혹. 하위(narrower) 자동 확장은 재현율↑처럼 보이나
정밀도를 무너뜨림 → 기본은 추천만. 자동확장은 **동의어 한정**.

**재사용 조건**: 정밀도가 중요한 도메인(법률·금융·의료 등) 검색. 관계가 풍부할수록 이 차등이 필수.

**prism-fs 적용**: 동의어만 `expand_query`로 BM25 병합, 관련/상위/하위/측정대상/공시영역은
`suggest_terms` 칩. 검수표 "병합금지(C)" 원칙과 정합.

## P3 — 설명 가능한 Graph-RAG

**문제**: 벡터 RAG는 블랙박스 — "왜 이 문서가 나왔는가"를 설명·감사 불가.

**기법**: 검색 경로를 **온톨로지 traversal로 명시**한다.
```
질의 → (동의어 자동병합) 개념 노드 → 타입드 엣지 traversal → 연결된 문서
     → provenance 동봉: {matched_via, path, relation_types, confidence, engine}
```
- 벡터(재현율) + 그래프(설명·정밀)를 **결합**. 그래프가 "왜"를 책임진다.
- 결과마다 결정론 근거를 붙여 감사 가능성·신뢰 확보.

**함정**: 그래프만으로 재현율 한계 → 임베딩과 hybrid 유지. 경로가 길면 설명 가독성 저하 → 1~2홉 우선.

**재사용 조건**: 규제·감사 대상 도메인, "근거 제시"가 요구되는 RAG. 의사결정 지원 도구.

**prism-fs 적용**: `concept_graph_demo.html` — 질의→개념(글로우)→경로 하이라이트→4사 주석 +
`engine:deterministic` provenance. 안전경계(숫자 AI 무경유) 정합.

## P4 — HITL 피드백 선순환 + 검수 게이트

**문제**: 지식베이스를 잡음 신호로 자동 갱신하면 **쓰레기 축적·드리프트**. 그렇다고 수동 편집만
하면 검색 결과와 단절.

**기법**: 검토 시점 **explicit 피드백** → `pending` → **사람 검수 게이트** → `active`. 2단계(캡처→승격).
- 한 번의 피드백이 **지식 그래프(P1)** 와 **평가 골든셋(P5)** 을 동시에 키운다.
- 게이트가 안전의 심장 — 자동 반영 금지.

**함정**: ① 암묵 신호(클릭/체류)는 잡음 → **explicit(👍/👎·정답지정·관계제안) 우선**. ② 자동 승격
유혹 → 사람 확정 필수. ③ 무한 stub 확장 → "재마주침/증거 기반"으로만 승격.

**재사용 조건**: 도메인 전문가가 검토 루프에 있는 시스템. 큐레이션 도구(validate/diff/apply+백업)
패턴과 결합.

**prism-fs 적용**: 검색 카드 피드백 → `search_feedback.json`(pending) → `curate_*` 도구로 승격 →
`concept_graph`/`golden_search.json`. 기존 회계동의어 검수표 프로세스의 자연 확장.

> 사상적 배경: Karpathy "interlinked notes"의 능동적 큐레이션 — LLM 자동생성이 아니라 **사람의 확정**이
> 지식을 정착시킨다. 검수 게이트 = 그 큐레이션 단계.

## P5 — 측정 우선 + 회귀 게이트

**문제**: 신뢰할 평가셋 없이는 "개선"을 믿을 수 없고, 튜닝이 조용히 회귀시킴.

**기법**: **골든셋 + eval 프로브**(hit@k·MRR·교차변형)를 변경 **전후로 실행**, 회귀는 거부. 골든셋은
P4 루프로 성장(placeholder → 확정셋).

**함정**: 작은 placeholder 골든셋의 만점을 과신. → 실사용 피드백으로 확정셋 확대가 전제.

**재사용 조건**: 검색·랭킹·추출 등 정확도를 튜닝하는 모든 시스템. 반영 전 게이트로 상시 가동.

**prism-fs 적용**: `scripts/eval_search.py --probe`. 실제로 A-2/B-3(사용자사전·청킹) 시도를 측정으로
회귀 포착(hit@1 1.0→0.88)해 **롤백** — 게이트가 작동한 증거.

## P6 — 정밀도-보존 쿼리 확장

**문제**: 동의어 확장을 임베딩 질의에까지 적용하면 의미 벡터가 흐려져 정밀도 하락.

**기법**: 동의어 확장은 **lexical/BM25 채널에만**, 임베딩 질의는 **원문 유지**. 확장 시 **단어 경계
인식**(예: '리스'가 '리스크'에 오확장 안 되게).

**재사용 조건**: 임베딩 + 어휘(BM25) 하이브리드 검색. P2와 짝.

**prism-fs 적용**: `synonyms.expand_query`는 BM25 코퍼스에만, `make_embedding`은 원질의. 한글+영숫자
경계 매칭.

---

## P7 — 핫패스 분리 (사전계산 → JSON)

**문제**: 대용량 파싱(예: 12MB XBRL)·임베딩을 요청 경로에서 하면 지연·자원 폭증.

**기법**: 무거운 계산은 **개발/배치 단계에서 사전 계산** → 셀/문서별 JSON 산출 → 런타임 엔드포인트는
**JSON만 읽음**(파싱·네트워크 0). 동일 코어를 런타임·배치가 공유해 스키마 드리프트 0.

**재사용 조건**: 결정론·읽기전용 산출물을 반복 조회하는 시스템. 폐쇄망·오프라인 우선.

**prism-fs 적용**: `fs_structured.json`·`xbrl_tagging.json`·`index.json`(schema=2) 모두 이 패턴.

## P8 — 폐쇄망 친화 그래프 시각화

**문제**: 그래프/네트워크 시각화 라이브러리는 보통 CDN/번들 의존 → 폐쇄망·에어갭에서 불가.

**기법**: **무의존** force-directed 레이아웃을 vanilla JS/SVG로 직접 구현(반발력+스프링+중심중력,
쿨링), 데이터 임베드, 외부 CDN 0. 무의존 자체가 폐쇄망 데모의 WOW.

**재사용 조건**: 에어갭·폐쇄망 데모, 의존성 최소화가 가치인 환경. 노드 수가 적당(수십~수백)할 때.

**prism-fs 적용**: `concept_graph_demo.html` — 직접 구현 force 레이아웃·드래그·타입별 엣지 스타일,
콘솔 에러 0·외부 CDN 0 검증.

## 재사용 체크리스트
- [ ] 기존 용어집/동의어가 있는가 → **P1**으로 무손실 레이어링(하위호환 폴백).
- [ ] 관계를 풍부하게 둘 것인가 → **P2** 차등 동작 필수(동의어만 병합).
- [ ] 설명·감사가 요구되는가 → **P3** provenance 동봉, 벡터와 hybrid.
- [ ] 전문가 검토 루프가 있는가 → **P4** explicit 피드백 + 검수 게이트(자동반영 금지).
- [ ] 정확도를 튜닝하는가 → **P5** 회귀 게이트 상시 가동, 골든셋을 실사용으로 확대.
- [ ] 임베딩+어휘 하이브리드인가 → **P6** lexical만 확장, 임베딩 원질의.
- [ ] 무거운 선계산이 있는가 → **P7** 오프라인 산출 JSON, 런타임 읽기전용.
- [ ] 폐쇄망 시각화가 필요한가 → **P8** 무의존 직접 구현.

## 안티패턴 (하지 말 것)
- 모든 관계로 쿼리 자동확장 (P2 위반 — 오확장).
- 동의어를 임베딩 질의에 병합 (P6 위반 — 정밀도 하락).
- 피드백 자동 반영 (P4 위반 — 드리프트·쓰레기).
- 측정 없이 튜닝 (P5 위반 — 조용한 회귀).
- 합성어를 단일 토큰 고정해 교차변형 BM25 중첩 약화 (prism-fs 실측 회귀 — A-2 사례).
