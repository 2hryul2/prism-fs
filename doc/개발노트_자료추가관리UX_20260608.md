# 개발노트 — 라이브러리 자료추가/관리 UX 개선

## 2026-06-08 10:40 작업 내역

세 가지 기능 구현(plan: `~/.claude/plans/1-pdf-velvet-ullman.md`).

### Feature A — "➕ 분기/연간 자료 추가" 모달 팝업
- 인라인 `#add-panel`(DART 수집) + 상단 `PDF 업로드` 탭을 **모달 1곳**으로 통합.
- `#add-modal`(+백드롭) 안에 서브탭 2개: `🛰️ DART 수집`(add-*), `⬆️ PDF 업로드`(up-*).
- 상단 `PDF 업로드` 탭 제거(`data-tab=upload` 버튼·`#tab-upload` 페인·중복 `dc-*`/`doCollect` 삭제).
- JS: `openAddModal/closeAddModal/showAddSub`, 백드롭 클릭·Esc 닫기. `add-toggle`·`onEmptyCellClick`·`openAddPanel`/`toggleAddPanel`(별칭) → 모달 위임. `populatePeriodSelects`에서 dc-* 제거.

### Feature B — 문서별 삭제 + 셀 전체 삭제
- 백엔드: `DELETE /api/library/{company}/{period}/doc/{doc_type}` 추가(기존 whole-cell DELETE 유지).
  - 해당 doc_type PDF·인덱스·원본 사본만 제거 → `_strip_doc_fields`로 카탈로그 해당 필드만 제거.
  - 남은 PDF 0 + 재무데이터 없음 → 셀 전체 제거(`scope:"cell"`), 아니면 `scope:dt`.
  - **접두 충돌 주의**: `review_sep_*`도 `review_`로 시작 → review 삭제 시 `not startswith("review_sep_")` 가드(없으면 연결 삭제가 별도 데이터까지 삭제).
- 프론트: `renderMatrix` 각 pill에 ✕(`.dp-del`, `deleteDoc`), 셀 우상단 🗑(`.cell-del`, `deleteCell`). `confirm()` 후 fetch → `SELECTED` 정리 → `loadLibrary`. `event.stopPropagation()`로 선택 토글과 분리.

### Feature C — 업로드 PDF 자동 감지·보정 제안
- 백엔드(순수 함수): `detect_company/doc_type/period_from_text` + `detect_pdf_meta_from`, `POST /api/library/detect`(파일명 우선 → 부족 시 1페이지 텍스트 폴백, 디스크/카탈로그 무변경).
- 휴리스틱: 회사(부분일치, KB금융 변형), 날짜 `(YYYY[.MM])`(월 옵션), 기간(사업보고서→FY/반기→Q2/분기→월 Q1·Q3/불명 Q3), 문서유형(연결검토→review/검토·감사&!연결→review_sep/분기·반기·사업&!검토·감사→report/연결&검토·감사→review).
- 프론트: `#up-file` change → `detectFromFilename`(JS 미러) → 불일치 시 "🔎 감지 … [자동 채움]/[무시]" 배너, 일치 시 "✓", 미감지 숨김. `applyDetect`로 드롭다운 보정(옵션 존재 시만).
